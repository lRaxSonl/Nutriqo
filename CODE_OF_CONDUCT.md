# Code of Conduct

## Our Commitment

**Nutriqo** project is committed to providing a welcoming and inclusive environment for all contributors, regardless of:
- Age, body size, disability
- Ethnicity, sex characteristics
- Gender identity and expression
- Level of experience
- Education
- Socio-economic status
- Nationality, personal appearance
- Race, religion
- Sexual identity and orientation

We are dedicated to making participation in this community a harassment-free experience for everyone.

## Our Standards

Examples of behavior that creates a positive environment include:

✅ **Do**:
- Use welcoming and inclusive language
- Be respectful of differing opinions, viewpoints, and experiences
- Accept constructive criticism gracefully
- Focus on what is best for the community
- Show empathy towards other community members
- Help and support other developers
- Acknowledge good ideas and give credit
- Ask clarifying questions before judgment

❌ **Don't**:
- Use sexualized language or imagery
- Make unwelcome sexual advances
- Make attacks personal or based on identity
- Trolling, insulting/derogatory comments
- Public or private harassment
- Publishing others' private information
- Other conduct which could reasonably be considered inappropriate
- Discrimination based on protected characteristics
- Intimidation or threats

## Enforcement

### Reporting

If you witness or experience unacceptable behavior:

1. **Document** the incident (date, time, participants, description)
2. **Report** to maintainers via:
   - Email: Clearly marked as "Code of Conduct Report"
   - GitHub: Private security advisory
   - Discord: (if community channel exists)

3. **Include**:
   - Description of incident
   - Participants involved
   - Context and impact
   - Any supporting evidence

### Confidentiality

All reports are handled confidentially. The identity of the reporter will not be disclosed without consent.

### Investigation & Response

- Maintainers will investigate all complaints
- Response within 48 hours
- Actions taken based on findings
- Follow-up communication with reporter

### Consequences

Violations may result in:

| Severity | Action |
|----------|--------|
| Minor | Warning + education |
| Moderate | Temporary suspension (7-30 days) |
| Severe | Permanent ban from community |

## Examples

### Acceptable Behavior

**During Code Review**:
```
"I see an alternative approach here. Have you considered using 
XYZ? It might be cleaner because..."
```

**In Discussions**:
```
"I disagree, but I understand your perspective. Here's why I think 
differently: ..."
```

**Giving Feedback**:
```
"Great contribution! I have a small suggestion in line 42. Would you 
consider refactoring it this way?"
```

### Unacceptable Behavior

❌ "This code is stupid"  
❌ "Only an idiot would do this"  
❌ "Women/men can't code"  
❌ "Go away, you're not wanted here"  
❌ "This is a terrible idea, you're wrong"  

## Appeals

If you believe you were unjustly sanctioned:

1. Request appeal in writing to maintainers
2. Explain your perspective
3. Appeal committee reviews decision
4. Decision made within 14 days

## Scope

This Code of Conduct applies to:

- Project repositories (Issues, Pull Requests, Discussions)

## Communication Guidelines

### Before Asking for Help

1. **Search existing issues/discussions** - Your question might be answered
2. **Check documentation** - README, ARCHITECTURE, QUICKSTART
3. **Search Stack Overflow** - Similar problems might be documented
4. **Review error messages** - Error text often tells you the solution

### Asking Good Questions

✅ **Good Example**:
```
Title: Photo analysis photo failing with 401 for premium users

Environment:
- Next.js 16.1.6
- Node 18.17
- Local development

Steps to reproduce:
1. Login with premium account
2. Upload food photo via UI
3. See error in console

Error:
"OPENAI_API_KEY is not set"

I've tried:
- Checking .env.local contains key
- Restarting dev server
- Clearing node_modules

What am I missing?
```

❌ **Poor Example**:
```
Photo analysis doesn't work
```

### Constructive Criticism

✅ **Helpful**:
- "I found a simpler approach. Would this work better?"
- "This might have performance issues with large datasets"
- "The test coverage could be improved for this function"

❌ **Unhelpful**:
- "This is wrong"
- "This code sucks"
- "Only stupid people would write this"

## Inclusive Language

We are committed to using language that is inclusive and welcoming:

- Use they/them pronouns unless otherwise specified
- Avoid gendered language (use "developer" not "he/she")
- Use "person" or "individual" not "man" or "girl"
- Avoid idioms that might not translate across cultures

## Accessibility

We value accessibility. Please:
- Use alt text for images
- Provide transcripts for videos
- Use clear, simple language
- Test content with screen readers
- Ask if you need accommodations
