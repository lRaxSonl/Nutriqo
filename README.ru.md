## 🎯 Обзор

**Nutriqo** — это современное приложение для отслеживания питания, которое использует ИИ для анализа фотографий пищи и помощи пользователям в поддержании сбалансированной диеты. Приложение обеспечивает анализ питания в режиме реального времени (белки, жиры, углеводы), отслеживание целей и функции премиум-подписки с интеграцией Stripe.

### Ключевые возможности

- 🤖 **Анализ фотографий с ИИ**: Использует GPT-4 Vision от OpenAI для анализа фотографий пищи и извлечения данных о питательной ценности
- 📊 **Расширенная статистика**: Отслеживание БЖУ (белков, жиров, углеводов) с подробными дневными и недельными отчетами
- 💳 **Премиум-подписки**: Интеграция Stripe для премиум-функций за $4,99/месяц
- 🔐 **Безопасная аутентификация**: OAuth (Google) + Email/пароль с NextAuth.js
- 👮 **Панель администратора**: Управление доступом на основе ролей
- 🌗 **Темные и светлые темы оформления**: Переключение между темами с сохранением пользовательских настроек
- 📱 **Адаптивный дизайн**: Мобильный приоритет с TailwindCSS
- ⚡ **Высокая производительность**: Next.js 16 с Turbopack для молниеносных сборок

### Архитектурный паттерн: FSD (Feature-Sliced Design)

Этот проект следует архитектуре **Feature-Sliced Design**:
- **entities/**: Основные бизнес-объекты (User, Food и т.д.)
- **features/**: Независимые бизнес-функции
- **shared/**: Общий код без бизнес-логики
- **widgets/**: Сложные UI-компоненты, объединяющие несколько функций

## 🚀 Начало работы

### Предварительные требования

- Node.js >= 18.17
- npm или pnpm
- Экземпляр PocketBase (локальный или размещенный)
- Ключ API OpenAI (для анализа фотографий)
- Учетные данные Google OAuth
- Ключи Stripe API

### Установка

1. **Клонируйте репозиторий**
   ```bash
   git clone https://github.com/lRaxSonl/Nutriqo.git
   cd Nutriqo/nutriqo-app
   ```

2. **Установите зависимости**
   ```bash
   npm install
   # или
   pnpm install
   ```

3. **Подключите переменные окружения** (см. [Конфигурация](#конфигурация))
   ```bash
   cp .env.example .env.local
   # Отредактируйте .env.local с вашими учетными данными
   ```

4. **Запустите сервер разработки**
   ```bash
   npm run dev
   # Сервер запустится на http://localhost:3000
   ```

5. **Получите доступ к приложению**
   - Откройте http://localhost:3000
   - Зарегистрируйтесь или войдите через Google OAuth
   - Начните отслеживать свое питание!

### Сборка для производства

```bash
npm run build      # Создать оптимизированную сборку
npm start          # Запустить сервер производства
```

## ⚙️ Конфигурация

### Требуемые переменные окружения

Создайте `.env.local` в директории `nutriqo-app/`:

```env
# === POCKETBASE (База данных и аутентификация) ===

# === Конфигурация NEXTAUTH ===

NEXTAUTH_SECRET=generate-with: openssl rand -base64 32

# === GOOGLE OAUTH ===

GOOGLE_CLIENT_SECRET=your-client-secret

STRIPE_SECRET_KEY=sk_test_...

STRIPE_WEBHOOK_SECRET=whsec_...

# === OPENAI (Анализ фотографий пищи) ===
```

## 📡 Документация API

### Базовый URL
```
http://localhost:3000/api
```

### Аутентификация
Большинство эндпоинтов требуют заголовок `Authorization: Bearer <token>`.

### Ключевые эндпоинты

#### Управление пищей
- `POST /api/food/add-entry` - Добавить новую запись о пище
- `GET /api/food/get-entries` - Получить записи пищи пользователя
- `DELETE /api/food/delete-entry` - Удалить запись о пище
- `POST /api/food/analyze-photo` - Анализировать фотографию пищи (Premium)

#### Управление целями
- `POST /api/goal/set-daily` - Установить ежедневные цели питания
- `GET /api/goal/get-daily` - Получить сегодняшние цели
- `GET /api/goal/get-all` - Получить все цели пользователя
- `PATCH /api/goal/update-daily` - Обновить ежедневные цели
- `POST /api/goal/finish-daily` - Завершить день и получить статистику
- `DELETE /api/goal/delete-daily` - Удалить конкретную цель

#### Платежи
- `POST /api/payment/checkout` - Создать сессию оформления заказа Stripe
- `POST /api/payment/activate-subscription` - Активировать подписку после оплаты

#### Администвор (требуется роль администратора)
- `GET /api/admin/dashboard` - Статистика панели управления
- `GET /api/admin/users` - Список всех пользователей
- `PATCH /api/admin/users/[id]` - Обновить роль пользователя
- `DELETE /api/admin/users/[id]` - Удалить пользователя

Полные описания эндпоинтов см. в файлах `src/app/api/*/route.ts`.

## 🏗️ Архитектура

### Поток аутентификации

```
Вход пользователя → Сессия NextAuth
    ↓
Провайдер OAuth/учетных данных
    ↓
Обратный вызов SignIn (создает/находит пользователя PocketBase)
    ↓
Обратный вызов JWT (синхронизирует роль, статус подписки)
    ↓
Обратный вызов Session (распространяет на фронтенд)
    ↓
Крючок useSession() в клиентских компонентах
```

### Стратегия синхронизации данных

```
PocketBase (Источник истины)
    ↓
JWT токен (Временный кэш)
    ↓
Сессия NextAuth (Доступ на фронтенде)
    ↓
Состояние React компонента
```

### Система подписки

```
Пользователь нажимает "Подписаться"
    ↓
Создается сессия оформления заказа Stripe
    ↓
Пользователь завершает платеж
    ↓
Эндпоинт activate-subscription
    ↓
subscriptionStatus устанавливается на 'active' в PocketBase
    ↓
Обратный вызов JWT обновляет и синхронизирует сессию
    ↓
Paywall исчезает, функции разблокируются
```
